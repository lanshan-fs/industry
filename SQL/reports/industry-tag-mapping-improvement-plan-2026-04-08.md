# 行业标签映射改进方案

日期：2026-04-08

## 当前基线

- 企业总数：25972
- `category_industry_company_map` 映射总行数：14402
- 至少命中 1 个行业分类的企业数：9578
- 当前覆盖率：36.9%
- 多重命中企业数：3579
- 有 `industry_belong` 文本但未命中的企业数：16476
- 含“医疗 / 医药 / 医院”等明显医疗关键词但仍未命中的企业数：14109
- 当前正式“行业标签”写入 `company_tag_map` 的行数：0

## 现状问题

1. 当前一级行业分类映射采用“四字段拼接 + 任一关键词命中即落表”的保守规则，来源见：
   - `SQL/scripts/rebuild_category_industry_company_map.py`
   - `SQL/data/category_industry_company_rules.json`
2. 命中逻辑没有字段权重、主次分类、冲突压制、低置信拦截。
3. “医院 / 门诊部 / 健康管理 / 技术服务 / 西药 / 中药 / 检验检测”等泛词会跨多个分类重复触发。
4. 泛医疗科技服务商、消费医疗机构、零售流通企业之间的误判较多。

## 本轮落地目标

1. 把映射逻辑从“命中即写入”升级为“打分后写入”。
2. 采用更接近人类专家的“主标签 + 副标签”判定，而不是单标签或无约束多标签。
3. 先提升精确率，再逐步扩大覆盖率。
4. 规则格式尽量兼容现有 `category_industry_company_rules.json`，便于后续继续调词。

## 落地策略

### 1. 打分式分类

- 使用字段权重：
  - `industry_belong` 最高
  - `company_name` 次高
  - `qualification_label` 中等
  - `business_scope` 最低
- 同一关键词在高置信字段命中时分更高。
- 仅在 `business_scope` 中出现的弱信号，不再直接入表。

### 2. 直达映射优先

- 对部分高确定性的国标行业名称做直达映射，例如：
  - `西药零售 -> 医疗零售`
  - `医疗用品及器材零售 -> 医疗零售`
  - `医疗用品及器材批发 -> 医药商业 / 流通`
- 这类命中优先级高于普通关键词。

### 3. 弱词降权

- 对“医疗 / 健康 / 技术服务 / 销售 / 软件开发 / 门诊部 / 医院 / 中药 / 西药”等高歧义词进行降权处理。
- 弱词只有与强词组合出现时才允许通过阈值。

### 4. 主标签 / 副标签机制

- 每家企业保留 1 个主标签。
- 允许再保留 1 到 2 个副标签，但必须满足：
  - 分数接近主标签
  - 有强证据或高置信行业归属支撑
  - 与主标签具备可共存关系
- 不允许互斥标签同时出现。

### 5. 冲突压制

- “命中多个词”不再等于“应该有多个标签”。
- 对互斥分类建立排斥关系，例如：
  - 医药商业 / 流通 与 医疗零售 不同时作为双标签保留
  - 化学制药 / 生物制品 / 中药原则上只保留最强主标签
  - 严肃医疗 / 消费医疗 / 互联网医疗 / 第三方中心之间默认只保留最强服务标签，除非存在明确共存证据

### 6. 验证标准

- 抽样精确率不低于 60%
- 多标签结果应具备可解释性，而不是简单回到“多重命中”
- 明显医疗信号企业的漏判率下降

## 本轮实施顺序

1. 改造 `SQL/scripts/rebuild_category_industry_company_map.py`
2. 保持 `SQL/data/category_industry_company_rules.json` 兼容可调
3. 重建并输出报告
4. 对重建结果做抽样复核
5. 结果稳定后，再考虑把行业分类同步为正式“行业标签”

## 后续可继续优化的方向

1. 为规则文件增加字段级命中配置，而不是只保留 `include_any / exclude_any`

4. 稳定后再把 `category_industry_company_map` 与 `company_tag_map` 的行业维度统一
